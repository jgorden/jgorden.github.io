Arrays and Hashes are two ways of storing information in Ruby. I never thought of them as similar, but upon closer inspection it's true their uses are remarkably redundant. Regradless, let us explore the distinction between the two.

I like to think of an array as an ordered list. An array, for the record, is an object that house some number of elements between two squre brackets. A couple examples:

Array = [1, 2, 3]

greetings = ["hello", "hola", "ni hao"]

Pretty simple! We name an object and point it to our array, and an array is born. What gives an array it's 'order' is the way it indexes by numbers. What if, say, I wanted to call up the first item in my 'Array' array. I could put in the following:

Array[0]

and ruby would spit out '1', or the first item in the array. Arrays are indexed with numbers starting at zero, increasing by for each item. So in the 'greetings' array, greetings[0] is hello, greetings[1] is hola, and ni hao would be greetings[2]. Easy! Because of this simple indexing arrays are very easy to visualize and work with. However, sometimes we want an index that is more specific, and in these cases we use a Hash.

Hashes are basically arrays with custom indexes. The notation is a bit different, so let us use our 'greetings' array from before as an example, except this time as a hash:

greetings = {

      :English => "hello",

      :Spanish => "hola",

      :Mandarin => "ni hao"

}

The first thing that you may notice is that instead of square brackets we have curly braces. Between the braces we have what are called key value pairs. The first bit, preceded by a colon, is the key, or our index. We can make these anything we want! We then have an arrow that points to our value, in this case our greetings represented as strings. So how do we access the values? The same way with arrays! So greetings[:English] will return 'hello' and so on. The advantage here is we can order our values in a way that makes sense, such as the language and the word.

Numbers work fine as indexes, but they are lacking when it comes to description. For times when having a set order for your items is useful, an array would be a good choice. For when you want to group the data in a way that may makes more semantic sense, use a hash.
