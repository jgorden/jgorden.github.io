Ruby Classes! So convenient. A class in ruby has a variety of advantages, including it's own variables and methods. I like to think of a class as a shell of sorts, or perhaps a mold for creating new objects of the same type. A class is an object that's use is to create new objects. Objects that belong to this class type will inherit instance variables and methods the moment they are born, potentially saving a lot of time. Lets run through an example:

class Cat
    def initialize(name, breed, age)
        @name = name
        @breed = breed
        @age = age
    end

    def talk
        p "Meow!"
    end

    attr_accessor :name, :age
    attr_reader :breed
    attr_writer :fav_food

    def feed(food)
        if food == @fav_food
            print "*chomp*"
            sleep 0.5
            puts " *chomp*"
            sleep 1.0
            puts "purr purrrrr"
        else
            3.times do
                print "."
                sleep 1.0
            end
            puts "blegh"
        end
    end
end

Let's break this down. We have started by opening a new class, in this case Cat. This class will act as a sort of momma cat, her children will be full fledged Cats with her contained features, but with slight individual variations. To express these variations we define an initialization. This method makes it so when a new Cat is created, it will need three arguments passed with it, those being name, breed, and age. We can try this! How about something like:

    garfield = Cat.new("Garfield", "Tabby", 37)
    => #<Cat:0x000000022c81f0 @name="Garfield", @breed="tabby", @age=37>

Notice what gets returned! There is a Cat object with it's ID, and variable assignments, except these variables have an @ sign preceding them. These are instance variables, we previously set them in the initialization. They are called 'instance' variables because every instance of the class Cat will have it's own individual value. This is in contrast with class variables(marked by @@), which will have the same value for all objects that belong to the class.

The next line shows an example of a simple instance method. We define a method 'talk', and supply the code to be run when called. This would run as follows:

    garfield.talk
    => "Meow!"

Simple enough! To make instance methods out of our instance variables, we can use the shorthand attr_accessor, reader, or writer. attr_reader does what you might think, it creates a method that allows us to read a value. attr_writer is also fairly intuitive, it allows you to assign a value outside of the class code, but excludes the reading privledges. attr_accessor is simply a combination of both, one is allowed to both call the method to read the value or set a new value. Notice that name and age are set as attr_accessor. Lets say our friend Garfield is celebrating a birthday. We can change his age and check the value as follow:

    garfield.age
    => 37
    garfield.age = 38
    => 38
    garfield.age
    => 38

Again, simple! I set 'breed' to attr_reader, you can imagine what this does. It even makes real life sense, as it isn't hard to check a cat's breed, but you would be hard pressed to change it. What about attr_writer though, where we have the yet undefined fav_food. Let's try supplying a value.

    garfield.fav_food = "lasagna"
    => "lasagna"

Alright, Garfield now has a favorite food of lasagna. Let's check this now with the feed method provided:

    garfield.feed("lasagna")
    *chomp* *chomp*
    purr purrrrr
    garfield.feed("salad")
    ...blegh

Great! But cat's are notoriously finicky eaters, and it would make sense to be able to rewrite the favorite food. This can be done easily, in fact it is the same as what we did with age before:

    garfield.fav_food = "tuna"
    => "tuna"
    garfield.feed("lasagna")
    ...blegh
    garfield.feed 'tuna'
    *chomp* *chomp*
    purr purrrrr

Happy cat. What we have covered here is creating a new class and supplying instance variables and methods that the new obects of the class can utilize. Instance variables are marked by an @, and the methods can be created normally by defining them or using the shorthand attr_ notation. We did not cover class variables and methods, but it could be useful to note that class variables are marked with a double @@, and the methods are defined with a self.method notation. These variables and methods operate at the class level, so if we had a Cat class method it would be called with Cat.method. Making classes allows you to create an endless number of new, similar, objects with attributes already built in. It is, indeed, a very powerful tool.
